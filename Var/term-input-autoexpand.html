--- title: "term-input-autoexpand" layout: doc --- term-input-autoexpand is a variable defined in `term.el'.
Its value is 
nil

  • This variable's value is permanent if it is given a local binding.

Documentation:
If non-nil, expand input command history references on completion.
This mirrors the optional behavior of tcsh (its autoexpand and histlit).

If the value is `input', then the expansion is seen on input.
If the value is `history', then the expansion is only when inserting
into the buffer's input ring. See also `term-magic-space' and
`term-dynamic-complete'.

This variable is buffer-local.

You can customize this variable.
